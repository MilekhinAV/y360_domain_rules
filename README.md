# Yandex 360 Domain Policies Updater

Этот репозиторий содержит Python-скрипт для автоматизированного и безопасного добавления новых правил маршрутизации писем (domain policies) в Яндекс 360.

Основано на основе публичного документации API Яндекс 360 "Разрешенные и запрещенные отправители" - https://yandex.ru/dev/api360/doc/ru/ref/DomainPolicies/

Специфика API Яндекс 360 такова, что метод добавления правил (`PUT`) полностью перезаписывает существующий массив. Чтобы не потерять старые правила при добавлении новых, скрипт реализует безопасный сценарий работы со списками.

## 🛠 Как это работает (Логика скрипта)

Скрипт выполняет 4 последовательных шага:
1. **GET-запрос:** Получает актуальный список всех существующих правил маршрутизации.
2. **Проверка и Дополнение:** Проверяет, нет ли уже правила с таким именем (во избежание дублей), и добавляет новое правило в конец полученного массива.
3. **PUT-запрос:** Отправляет обновленный массив правил обратно на сервер. Существующие правила остаются нетронутыми, новое — добавляется.
4. **Контрольный GET-запрос:** Запрашивает итоговый список правил и выводит его в терминал в красивом JSON-формате для визуальной проверки.

## 📋 Требования

Для работы скрипта потребуется **Python 3.6+** и библиотека `requests`.

Установите зависимости, выполнив команду:
```
pip install requests
```

## 🚀 Настройка и использование

1. Клонируйте репозиторий или скачайте файл скрипта.
2. Откройте скрипт в любом текстовом редакторе и заполните конфигурационный блок:
   - `ORG_ID` — ID вашей организации в Яндекс 360.
   - `OAUTH_TOKEN` — ваш OAuth-токен с правами администратора.
3. Отредактируйте словарь `NEW_RULE` под ваши нужды (укажите нужное имя, описание и домены для блокировки или разрешения).

```
# ================= Конфигурация =================
ORG_ID = "ваш_org_id"
OAUTH_TOKEN = "ваш_oauth_токен"
```

4. Запустите скрипт:
```
python main.py
```
*(замените `main.py` на фактическое имя вашего файла)*

## 💡 Особенности работы фильтра `domain_filter`

В скрипте используется маска подстановки `*`. Например, значение `*.com` в правиле `reject`:
- **Блокирует** письма, где домен отправителя заканчивается на `.com` (например, `user@gmail.com`).
- **Пропускает** письма, где `.com` является частью имени пользователя до символа `@` (например, `mat.ru.com@yandex360.ru`), так как фильтр проверяет только доменную часть адреса.
